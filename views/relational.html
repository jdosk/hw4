What are Relational Databases?

Relational databases are databases that store information in relation to each other. Data is sorted and stored on tables of rows and columns. Each table will have a primary "key" that is used to identify information, as well as allow tables to relate to each other. In this way, relational databases can store and organize data that is easy to manipulate and gather. However, relational databases scale vertically, meaning they rely soley on a single server to manage the database, so increases in data will also require an increase in the server capabilities in order to keep performance levels the same.

For example, the image above contains three tables in a database. The first (top left) table contains unique identifiers: itemid, and orderid. itemid is the primary key, or the uniques identifier for each row in the table. As we can see orderid is also in the bottom table. These two tables are linked, or related, by the common identifying key.

Popular relational databases:

  • MySQL
  • PostgreSQL
  • Oracle DB
  • SQL Server
  • SQLite
